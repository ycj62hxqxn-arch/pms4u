# SET Compliance Model (v1.0)

This document defines the compliance conditions required by SET.

## Compliance Rule

A system is SET-compliant if and only if:

- Execution authority governs all transitions.
- No bypass path exists.
- Verification cannot trigger execution.
- Invariance cannot authorize execution.

## Evidence

SET compliance SHOULD be supported by:

- Transition logs
- Authority decision records
- Validation proofs

## Status

This model is normative for the public standard surface.
